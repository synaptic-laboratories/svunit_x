## What changed between x0.2.0 and x0.3.0

The six-target boundary, the upstream sync commit, and the Vivado
`synth-sim-full` profile are unchanged. x0.3.0 adds four opt-in developer
conveniences and one hardening fix:

- `--reuse-build` / `--xsim-reuse-build` — SHA256-keyed build-artefact cache
  for qrun, modelsim, and xsim when inputs are unchanged. Defaults off; the
  deterministic sign-off path never takes the cache.
- `--sim-debug-level {none,low,medium,med,high,all}` — normalised simulator
  debug level forwarded to supported back ends; plus `SVUNIT_SIM_DEBUG_LEVEL`
  env default.
- `--sim-runtime-stats` — request simulator runtime statistics when the
  back end supports them.
- `--xsim-run-mode {separate,standalone}` — xsim run shape; `separate`
  invokes `xsim`, `standalone` invokes `xelab -standalone -R`.
- xsim run-log hardening — SVUnit now rejects `-r_arg -nolog` /
  `-r_arg --nolog` and greps the xsim run log for startup-failure patterns
  that xsim has been observed to report while returning exit code 0.

The new switches are tested by the per-target regression suites and contribute
the additional pass counts reported below versus the x0.2.0 session.
