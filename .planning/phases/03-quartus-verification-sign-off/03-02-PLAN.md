```bash
#!/usr/bin/env bash
# 03-reproduce.sh — Reproducibility companion to 03-sign-off.md (D-07).
# Contract: drives svunit-certify-all, captures 5 per-target run-ids, writes a
# predictable output file the sign-off doc can template against. Does NOT author
# or commit the sign-off doc — that stays a human-in-the-loop step so gap-matrix
# and residuals sections can be re-judged against the new run.
#
# Usage:
#   bash .planning/phases/03-quartus-verification-sign-off/03-reproduce.sh [--out PATH]
#
# Preconditions:
#   - cwd must be the SVUnit repo root (Setup.bsh present)
#   - license files + container images + ARTEFACTS_ROOT present (see preflight below)
#   - nix, podman, jq, git on PATH
#
# Output:
#   - /tmp/svunit-reproduce-runids.<timestamp>.tsv  (default; overridable via --out)
#     Columns (TSV): target  run_id  qualification_status  tests_passed  tests_failed  tests_skipped  pytest_filter  svunit_commit
#   - Exit code 0 iff svunit-certify-all exit 0 AND 5 distinct run-ids captured AND all status=PASS.

set -euo pipefail

ARTEFACTS_ROOT="/srv/share/repo/sll/g_sll_infra/g_sll_infra_dev_001/g_ext_tools_qualified/g_svunit_x/r_v3_38_1_x0_2_0_artefacts"
LICENSE_DIR="/srv/share/repo/sll/g_sll_poc/g_2026/ContainerPlayPen/launch"
EXPECTED_TARGETS=(
  "quartus-23-4-qrun"
  "quartus-23-4-modelsim"
  "quartus-25-1-sim-only-qrun"
  "quartus-25-1-sim-only-modelsim"
  "verilator-5-044"
)
EXPECTED_IMAGES=(
  "localhost/quartus-pro-linux:23.4.0.79"
  "localhost/quartus-pro-linux:25.1.1.125-sim-only"
)
STAMP="$(date -u +%Y%m%d-%H%M%S)"
OUT="/tmp/svunit-reproduce-runids.${STAMP}.tsv"

# --- flag parsing ---
while [[ $# -gt 0 ]]; do
  case "$1" in
    --out) OUT="$2"; shift 2 ;;
    -h|--help)
      grep '^# ' "$0" | sed 's/^# //'; exit 0 ;;
    *) echo "ERROR: unknown flag: $1" >&2; exit 2 ;;
  esac
done

# --- preflight ---
echo "[preflight] cwd must be repo root..."
test -f Setup.bsh || { echo "FAIL: Setup.bsh not found — run from SVUnit repo root" >&2; exit 2; }

echo "[preflight] license files..."
for f in quartus_license.dat questa_license.dat; do
  test -f "$LICENSE_DIR/$f" || { echo "FAIL: missing $LICENSE_DIR/$f" >&2; exit 2; }
done

echo "[preflight] container images..."
for img in "${EXPECTED_IMAGES[@]}"; do
  podman image exists "$img" || { echo "FAIL: missing podman image $img" >&2; exit 2; }
done

echo "[preflight] artefacts root..."
test -d "$ARTEFACTS_ROOT" || { echo "FAIL: ARTEFACTS_ROOT absent: $ARTEFACTS_ROOT" >&2; exit 2; }

echo "[preflight] nix flake shows 5 per-target apps + aggregate..."
count=$(nix flake show 2>&1 | grep -c 'svunit-certify' || true)
[[ "$count" -ge 6 ]] || { echo "FAIL: nix flake show reported only $count svunit-certify apps (expected >= 6)" >&2; exit 2; }

# --- snapshot pre-run run-id set (for collision detection) ---
BEFORE="$(mktemp)"
ls -1 "$ARTEFACTS_ROOT" | grep -v '^latest$' | sort > "$BEFORE"

# --- drive the regression ---
echo "[run] nix run .#svunit-certify-all (sequential 5 targets; 20-60 min)..."
set +e
nix run .#svunit-certify-all 2>&1 | tee "/tmp/svunit-certify-all.${STAMP}.log"
CERT_RC=${PIPESTATUS[0]}
set -e

# --- derive the 5 post-run run-id dirs ---
AFTER="$(mktemp)"
ls -1 "$ARTEFACTS_ROOT" | grep -v '^latest$' | sort > "$AFTER"
NEW="$(comm -13 "$BEFORE" "$AFTER" || true)"
NEW_COUNT="$(printf '%s\n' "$NEW" | sed '/^$/d' | wc -l)"

echo "[verify] $NEW_COUNT new run-id dirs appeared after svunit-certify-all."

# --- write the predictable TSV the sign-off doc references ---
{
  printf 'target\trun_id\tqualification_status\ttests_passed\ttests_failed\ttests_skipped\tpytest_filter\tsvunit_commit\n'
  while IFS= read -r ri; do
    [[ -z "$ri" ]] && continue
    bi="$ARTEFACTS_ROOT/$ri/build-info.json"
    if [[ -f "$bi" ]]; then
      jq -r '[.target, .run_id, .qualification_status, .tests_passed, .tests_failed, .tests_skipped, .pytest_filter, .svunit_commit] | @tsv' "$bi"
    else
      printf 'UNKNOWN\t%s\tMISSING\t0\t0\t0\tMISSING\tMISSING\n' "$ri"
    fi

fi
```

```markdown
## Phase 3 — Quartus Verification & Sign-Off (2026-04-18)

### L3-01: Minute-granular run IDs collide when tools run fast
**Lesson:** Qualification tooling that builds run-id directories from `date -u +"%Y%m%d-%H%M"` (minute granularity) can silently overwrite artefacts when two sequential targets complete within the same UTC minute. The second target reuses the first's directory and wipes its `build-info.json`, `tests.xml`, etc.
**Evidence:** `scripts/qualification-helpers.sh` `qh_build_run_id --no-gpu-suffix`; `scripts/certify.sh:118-131` calls it per-target with no uniqueness guard.
**Generalization:** Any timestamp-derived identifier needs either (a) finer granularity, (b) a uniqueness guard (retry on collision), or (c) a target/tool discriminator baked into the ID. For existing tooling without these, the consumer must validate "expected N distinct IDs after N runs" post-hoc.
**Applicability for future sign-off rounds:** the reproducibility script MUST verify `distinct(run_id)` count matches expected target count before declaring sign-off green.

### L3-02: The `latest` symlink is overwritten per target, not per sign-off
**Lesson:** When an aggregator runs M targets sequentially and each target ends with `update_latest_symlink`, the final `latest` points at the LAST target — not the sign-off as a whole. Citations using `latest/` rot the moment another target runs, even within the same sign-off.
**Evidence:** `scripts/certify.sh:462` → `qh_update_latest_symlink`; live artefacts root showed `latest → <last-verilator-run>` after the three captured run-id dirs.
**Generalization:** "Latest" aliases are maintenance conveniences, not citation anchors. Any document claiming durable evidence must cite by full ID, not by any alias that the tool updates asynchronously.
**Applicability:** sign-off docs, audit reports, compliance records, CI pass-log references — always cite by immutable ID.

### L3-03: Xilinx-thematic greps that worked (Plan 1 heuristics)
**Lesson:** For auditing an upstream merge against fork-specific SystemVerilog conventions, these four greps (derived in Plan 1) reliably surface candidates:
- **Parser-safe queue typing (T1):** `grep -rnE 'typedef\s+\w+\s+\w+\[' | grep -v '\[\$\]'` surfaces typedefs lacking the `[$]` dynamic-array form.
- **Explicit `input` signatures (T2):** `git diff <merge-base>..HEAD -- <scope> | grep -E '^\+.*\bfunction\b' | grep -v 'input '` surfaces NEW function signatures lacking the `input` keyword.
- **`XILINX_SIMULATOR` ifdef guards (T3):** `grep -rn 'XILINX_SIMULATOR' <scope>` confirms preserved guard count pre- vs post-merge.
- **xsim runtime flags (T4):** `grep -nE 'xvlog|xelab|xsim\.dir|xvlog\.pb' bin/runSVUnit bin/cleanSVUnit` confirms retained LCU-01 markers.
**Evidence:** `03-RESEARCH.md` §Focus Area 1; `03-xilinx-thematics-audit.md` §Theme T1..T4 Findings.
**Generalization:** For any "fork-specific convention preserved across upstream merge" audit, the pattern is: (a) enumerate the convention's prior-art markers (what the fork author CONSISTENTLY did), (b) grep the merge-imported surface for the anti-pattern (what upstream code would naturally do WITHOUT the convention), (c) classify each hit by a fixed A/B/C scheme. The audit is reproducible because the greps are explicit.
**Applicability:** any future upstream catch-up where fork intent is documented in prior-art markers.

**Generalization:** For any cross-cutting audit, an explicit file/directory allowlist outperforms a directory glob because the anti-pattern is usually absence (something NOT in the glob that should be) and allowlists make absence auditable.
```
